# 人脸识别监控系统

基于 dlib ResNet 深度学习模型的人脸识别系统（准确率 99.38%）

## 快速开始

### 0. 测试摄像头（推荐先测试）
运行测试脚本检查摄像头是否可用：
```bash
# 双击运行
测试摄像头.bat

# 或命令行
..\vip3-django\Scripts\python.exe 测试摄像头.py
```

**如果看不到摄像头画面**，请检查：
- Windows 隐私设置 → 相机权限
- 摄像头是否被其他程序占用（关闭微信、QQ、Teams等）
- 设备管理器中摄像头驱动是否正常

### 1. 准备人脸照片
将要识别的人的照片放到 `image/` 目录：
```
image/
  ├── 张三.jpg    ← 文件名即为识别显示的姓名
  ├── 李四.jpg
  └── ...
```

**照片要求**：清晰的正面人脸，每人1-3张即可

### 2. 启动系统
双击运行：`启动项目.bat`

或命令行启动：
```bash
cd smartbox
..\vip3-django\Scripts\python.exe manage.py runserver
```

### 3. 访问系统
浏览器打开：http://localhost:8000/monitor/

## 摄像头配置

### 本地摄像头
在数据库或管理后台配置摄像头时，RTSP地址填写：
- `0` - 第一个本地摄像头
- `localhost` - 本地摄像头
- `1` - 第二个本地摄像头（如有多个）

### 网络摄像头
填写完整的RTSP地址：
```
rtsp://用户名:密码@IP地址:端口/路径
```

## 调整识别阈值

如需调整识别灵敏度，编辑 `smartbox/monitor/views.py` 第 285 行：

```python
if best_distance < 0.6:  # ← 调整此值
    name = best_match
```

- **0.4-0.5**：宽松，易识别，可能误识
- **0.6**：推荐（默认）
- **0.7-0.8**：严格，减少误识

## 常见问题

### Q: 摄像头打不开？

**1. 先运行测试脚本**
```bash
双击运行: 测试摄像头.bat
```

**2. 检查系统设置**
- Windows设置 → 隐私 → 相机 → 允许应用访问相机
- 设备管理器 → 图像设备 → 检查摄像头驱动

**3. 检查占用情况**
关闭可能占用摄像头的程序：
- 微信、QQ、钉钉、Teams
- 浏览器的视频通话
- 其他视频软件

**4. 尝试不同的摄像头索引**
如果笔记本有多个摄像头，在数据库中尝试：
- `0` - 第一个摄像头
- `1` - 第二个摄像头

**5. 重启服务**
修改配置后重启Django服务

**Q: 识别不准？**
- 添加更多不同角度的照片
- 确保照片清晰且光线良好
- 适当调大阈值（改为0.65-0.7）

**Q: 如何添加新人脸？**
1. 将照片放到 `image/` 目录
2. 文件名即为显示名字
3. 重启Django服务
